# InputScreening_Submitted - Quick Reference Card

## 📋 What Is This?

Permanent **immutable snapshot** table storing complete state of all Input Screening submissions.
- **One lot = One record** (with unique lot_id)
- **Partial splits create child lots** (independent, automatic)
- **Future modules use this, not live data**

```python
record = create_full_accept_submission(
    original_lot_id="LID001", batch_id="B123", original_qty=500,

    tray_capacity=20, active_trays_count=25, top_tray_id="NB-001",
    top_tray_qty=3, all_trays_json=[...], created_by=request.user
)
# Result: Single record, all qty accepted, all trays to accepted_trays_json
```

### Create Full Reject (With Reasons)
```python
from InputScreening.services_submitted import create_full_reject_submission
```

### Create Partial Split (250 Accept + 250 Reject)
```python
from InputScreening.services_submitted import create_partial_split_submission
accept_rec, reject_rec = create_partial_split_submission(
    original_lot_id="LID003", batch_id="B125", original_qty=500,
    accept_qty=250, reject_qty=250,
    accept_trays_json=[...], reject_trays_json=[...],
    accept_tray_count=16, reject_tray_count=16,
    ... # other fields
)
# Result: TWO child lots created atomically
# accept_rec.lot_id = "LID<NEW_UUID1>" (independent)
# reject_rec.lot_id = "LID<NEW_UUID2>" (independent)
# Both have is_child_lot=True, parent_lot_id="LID003"
```

## 🔑 Key Concepts

### Lot ID Format
```
LID + 12-char UUID hex = "LID1A2B3C4D5E6F"
Generated by: generate_lot_id()
```

### Full Accept
- ✅ Entire lot accepted
- `is_full_accept=True`
- `accepted_qty = original_qty`
- `rejected_qty = 0`
- All trays → accepted_trays_json

### Full Reject
- ❌ Entire lot rejected with reasons
- `is_full_reject=True`
- `accepted_qty = 0`
- `rejected_qty = original_qty`
- All trays → rejected_trays_json
- Reasons → rejection_reasons_json

### Partial Split (Most Complex)
- ⚠️ Partial accept (250) + Partial reject (250)
- Creates **2 INDEPENDENT child lots**
- Each child has own lot_id (new UUID generated)
- Both marked `is_child_lot=True`
- Store `parent_lot_id` for reference
- **Future modules use child lot_id, NOT parent**

## ⭐ Integration Pattern for Downstream

### Current Way (WRONG ❌)
```python
# DayPlanning or BrassQC
batch = ModelMasterCreation.objects.get(batch_id=batch_id)
qty = batch.total_batch_quantity  # ❌ Can change if split!
```

### Correct Way ✅
```python
# DayPlanning or BrassQC
from InputScreening.services_submitted import get_lot_metadata_for_downstream

metadata = get_lot_metadata_for_downstream(original_lot_id)
actual_lot_id = metadata['lot_id']  # May be different!
qty = metadata['qty']  # Locked in from submission
trays = metadata['trays']  # Locked in from submission

# Use actual_lot_id and qty - they're immutable now
```

## 🔐 Atomic Guarantee

When creating partial split:
```
create_partial_split_submission() runs inside @transaction.atomic()
├─ If accept_record.save() succeeds AND reject_record.save() succeeds
│  └─ COMMIT: Both children in DB
├─ If ANY error occurs
│  └─ ROLLBACK: Neither child in DB (zero half-saves)
```

## 🔧 Troubleshooting

| Problem | Solution |
|---------|----------|
| Lot not found | Check `is_active=True` - may be revoked |
| Wrong qty in next module | Call `get_lot_metadata_for_downstream()` instead of fetching ModelMasterCreation |
| Child lot not found | Parent was split - use child lot_id from metadata |
| Duplicate lot_id error | Extremely rare, has 3-retry fallback |
| JSON showing empty | Normal - use `.json` fields in code, not admin view |
