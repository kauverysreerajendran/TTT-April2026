# IQF Lot Resubmission Fix - IMPLEMENTATION SUMMARY

## Problem Statement
Lots returning from IQF with `send_brass_qc=True` were blocked from resubmitting to Brass QC with HTTP 409 Conflict error:
```
[QC ACTION] Duplicate blocked: lot_id=LID170420261814450002
Conflict: /brass_qc/api/action/
```

## Root Cause
The duplicate submission check at `_handle_submission()` line 1098 did not distinguish between:
- ❌ **Legitimate blocking**: Normal duplicate submission (same lot, same cycle) 
- ✅ **Should allow**: IQF resubmission (same lot ID, fresh cycle from IQF return)

## Solution Implemented

### Change 1: IQF Reentry Detection & Conditional Blocking
**File**: `Brass_QC/views.py`  
**Lines**: 1097-1112

```python
# ─── Duplicate Submission Check with IQF Reentry Exception ───
# IQF-returned lots (send_brass_qc=True) are isolated submissions, NOT duplicates
is_iqf_reentry = bool(stock.send_brass_qc)

existing = Brass_QC_Submission.objects.filter(lot_id=lot_id, is_completed=True).first()
if existing and not is_iqf_reentry:
    # Block duplicates (normal flow)
    logger.warning(f"[QC ACTION] Duplicate blocked: lot_id={lot_id}")
    return JsonResponse({...}, status=409)

# For IQF reentry: clear old submission record to allow fresh submission
if existing and is_iqf_reentry:
    logger.info(f"[QC ACTION] IQF reentry detected for lot_id={lot_id}, clearing old submission record (id={existing.id})")
    existing.delete()
    existing = None
```

### Change 2: Flag Reset After Successful Submission
**File**: `Brass_QC/views.py`  
**Lines**: 1437 + 1450

```python
stock.send_brass_qc = False  # ✅ Clear IQF reentry flag after processing

stock.save(update_fields=[
    ...
    'send_brass_qc',  # ✅ Include flag reset in update
])
```

## How It Works

### Decision Logic
```
IF lot.send_brass_qc = True (IQF-returned lot):
   AND old submission exists:
      → DELETE old submission record
      → Allow resubmission (fresh cycle)
ELSE IF lot.send_brass_qc = False (normal lot):
   AND old submission exists:
      → BLOCK with 409 Conflict (safety)
```

### Execution Flow
```
1. Check: is_iqf_reentry = bool(stock.send_brass_qc)
2. Query: existing = Brass_QC_Submission.filter(lot_id, is_completed=True).first()
3. Decision:
   - Normal lot (send_brass_qc=False) + existing → Block with 409 ✅
   - IQF lot (send_brass_qc=True) + existing → Delete old + Allow ✅
   - No existing → Allow (normal flow) ✅
4. After submission succeeds: send_brass_qc = False (flag reset)
```
